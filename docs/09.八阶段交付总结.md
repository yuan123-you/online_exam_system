## 在线考试系统 — 八阶段重构与功能扩展交付总结

### 项目概况

本项目对在线考试系统进行了全面的架构重构和功能扩展，涵盖前端（Vue 3 + TypeScript + Vite）、后端（Spring Boot 3.3.5 + Java 21）和数据库（MySQL）三层，共完成 8 个阶段的改造工作。原始项目已在 `project-backup/original` 目录完整备份。

---

### 第一阶段：项目结构重构与规范化

**前端改造：**

原项目所有业务逻辑集中在单个 `App.vue`（1276 行）中，无路由、无状态管理。重构后拆分为 25+ 个视图组件，引入 vue-router@4 和 Pinia@2 状态管理：

- `src/router/index.ts` — 路由配置，含导航守卫和角色权限控制
- `src/stores/app.ts` — Pinia 全局状态，完整迁移自 App.vue
- `src/views/` — 按角色分为 admin（5）、teacher（7）、student（4）+ Profile/Login
- `src/composables/useDeviceType.ts` — 设备检测组合式函数
- `App.vue` 精简至约 68 行（仅 router-view + 全局弹窗 + Toast）

**后端改造：**

原项目所有接口集中在单个 `ApiController.java`（1483 行）中。重构后拆分为 64 个 Java 文件，按企业级标准分层：

- `controller/` — 15 个 REST 控制器（Auth、Bootstrap、Admin、Teacher、Exam、Submission、WrongBook、Student、Ai、ChatHistory、PracticeSession、Recommendation、Quota、ClassAnalysis、Notification）
- `service/` — 18 个业务服务类
- `repository/` — 9 个数据访问层（对应 9 张数据库表）
- `entity/` — 9 个 Java Record 实体类
- `dto/` — 数据传输对象
- `config/` — 拦截器、CORS、安全头、RestTemplate 配置
- `common/` — 认证、JSON、响应构建工具类

已删除旧文件 `ApiController.java`，保留 `StoreService.java` 作为兼容门面。

---

### 第二阶段：响应式布局与多端适配

- 建立统一的 CSS 断点系统（`src/styles/breakpoints.css`）
- 实现三端适配：移动端（<768px）、平板（768-1024px）、桌面端（>1024px）
- 侧边栏在移动端切换为抽屉模式，支持触摸手势滑动打开/关闭
- 底部 iOS 风格标签栏适配移动端
- `useResponsiveLayout.ts` — 语义化布局决策
- `usePlatformAdapter.ts` — 跨平台抽象层
- 向 `public/styles.css` 追加约 724 行响应式样式

---

### 第三阶段：登录页体验优化

- 使用隐藏虚拟输入框彻底禁用浏览器自动填充
- 密码可见性切换按钮已验证正常工作
- 新增聚焦态高亮和加载旋转动画
- 输入框 `autocomplete` 属性设置为 `new-password` / `new-username`

---

### 第四阶段：数据隔离与权限体系

- 创建数据库迁移 `V2__data_isolation.sql`，添加 `created_by` 和 `deleted` 字段
- 实现 `DataIsolationService`，防止水平越权（同角色访问他人数据）和垂直越权（低权限访问高权限接口）
- `AuthInterceptor` 增强：校验 `X-User-Id` 对应用户是否真实存在（含 30 秒缓存）
- 所有查询接口均绑定当前用户 ID 过滤

---

### 第五阶段：唯一标识与业务逻辑改造

- 验证所有实体间关联均使用 ID（VARCHAR(64)）而非名称
- 确认不存在基于名称的查找逻辑
- 在 question、paper、exam 表上实现软删除（`deleted` 字段），支持回收站恢复
- 级联删除逻辑完善：删除试卷自动关联其下的考试和提交记录

---

### 第六阶段：新增 AI 功能

集成了智谱 AI GLM-4.7-Flash 大模型（OpenAI 兼容接口），实现三大 AI 功能：

- **AI 智能出题**（`AiQuestionGen.vue`）：根据知识点、难度、题型自动生成试题
- **AI 辅助阅卷**（`SubmissionReviewModal.vue` 增强）：对主观题自动评分并给出解析
- **AI 刷题练习**（`AiPractice.vue`）：基于错题本和薄弱知识点智能推荐练习题

后端 `AiService.java` 实现了真实的 GLM-4.7-Flash API 调用，含速率限制、结果缓存和优雅降级。API Key 通过环境变量注入，不硬编码在代码中。

---

### 第七阶段：存储配额与批量操作

- 题库配额 5000 题，错题本/考试记录各 1000 条
- 创建可复用的 `QuotaBar.vue` 配额进度条组件
- 后端新增 `QuotaController`（GET /api/user/quotas）
- 题库管理支持多选复选框 + 批量删除
- 错题本支持批量清理
- 超出配额时自动归档最旧记录

### 第八阶段：测试与安全审计

**安全审计**发现并修复 7 个漏洞：

| 编号 | 严重等级 | 问题描述 | 修复措施 |
|------|----------|----------|----------|
| SEC-001 | 严重 | AuthInterceptor 仅检查头部存在，不验证用户真实性 | 添加 `isUserValid()` 含 30 秒缓存 |
| SEC-002 | 严重 | AI API Key 硬编码在配置文件中 | 改为环境变量注入 `${AI_API_KEY:}` |
| SEC-003 | 高 | AI 解题接口缺少认证校验 | 添加用户空值检查 |
| SEC-004 | 高 | 试题恢复绕过所有权校验 | 添加数据库级所有权验证 |
| SEC-005 | 高 | 数据库密码硬编码 | 改为环境变量注入 |
| SEC-006 | 中 | 登录接口无暴力破解防护 | 添加 5 次/5 分钟锁定机制 |
| SEC-007 | 低 | 教师更新时可伪造 teacherId | 强制覆盖为当前登录用户 |

**测试报告**覆盖 38 个测试用例（TC-01 ~ TC-38），涵盖全部 8 个阶段。
