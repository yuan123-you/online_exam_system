# 04.用户手册

## 1. 环境要求

| 项目 | 版本要求 |
|------|----------|
| Java | JDK 21 或以上 |
| Node.js | 18.x 或以上 |
| MySQL | 8.x |
| 浏览器 | Chrome 或 Edge 最新版 |
| Maven | 3.x（后端构建） |

## 2. 启动步骤

### 2.1 数据库准备

```bash
# 导入数据库
mysql -u root -p < database.sql
```

### 2.2 启动后端

```bash
# 方式一：使用 npm 脚本
npm start

# 方式二：直接运行 JAR
java -jar backend\target\online-exam-backend-1.0.0.jar
```

后端默认运行在 `http://localhost:8080`。

如果 MySQL 密码不是默认的 `123456`，需要用环境变量覆盖：

```powershell
$env:MYSQL_USER="root"
$env:MYSQL_PASSWORD="你的MySQL密码"
npm start
```

### 2.3 启动前端（Vue 3 版本）

```bash
npm run dev:web
```

前端开发服务器默认运行在 `http://localhost:5173`，并自动代理 `/api` 请求到后端 8080 端口。

### 2.4 使用独立版（原生 HTML 版本）

Spring Boot 启动后直接访问 `http://localhost:8080` 即可使用内置的原生 HTML 版本，无需额外启动前端。

## 4. 角色操作说明

### 4.1 管理员

- 登录后可在左侧菜单进入数据总览、学生管理、教师管理、组织管理、系统日志、个人信息
- 在数据总览页查看学生数、教师数、题库量、考试场次统计和题库分布图表
- 在学生/教师管理页面点击"新增"按钮录入信息，或点击"批量导入"上传 Excel/CSV 文件
- 在组织管理中维护学院和班级信息
- 在系统日志中查看操作记录（按时间倒序）

### 4.2 教师

- 进入教学看板查看题目、试卷、考试、答卷数据概览
- 进入题库管理创建题目，支持单选、多选、判断、填空、简答、编程六种题型
- 进入试卷管理选择题目组卷，支持科目和关键字筛选，系统自动排除已被其他试卷占用的题目
- 进入考试管理发布考试，选择试卷、班级和时间范围
- 学生交卷后，在阅卷中心查看答卷详情，给简答题打分并提交阅卷
- 在成绩分析中查看成绩分布图表和考试通过率

### 4.3 学生

- 在待考考试中查看可参加的考试，点击"进入考试"开始答题
- 答题过程中左侧有题号导航，已答题目显示绿色标记，当前题目高亮
- 可点击"保存答卷"手动保存，系统每 30 秒自动保存
- 切屏时系统自动记录次数，超过上限会自动交卷
- 提交后在考试记录中查看成绩、排名和用时
- 错题本自动归集答错的题目，支持重做和移除
- 在个人信息中可修改登录密码

## 5. 常见问题

### 5.1 页面打不开

- 确认后端 `npm start` 已成功启动，检查 `http://localhost:8080/api/health` 是否返回正常
- 如果使用 Vue 3 版本，确认 `npm run dev:web` 已启动

### 5.2 登录失败

- 确认使用上方演示账号
- 确认 MySQL 数据库已正确导入（执行 `database.sql`）

### 5.3 修改密码后忘记

- 管理员可通过后端管理接口重置密码
- 也可直接在 MySQL 中更新对应用户的 password 字段

### 5.4 端口冲突

- 后端 8080 端口被占用：在 `application.yml` 中修改 `server.port`
- 前端 5173 端口被占用：在 `vite.config.ts` 中修改 `server.port`
- MySQL 3306 端口被占用：确认 MySQL 服务已正确安装

## 6. 数据备份

- 数据库数据可通过 `mysqldump` 命令导出备份
- 独立版 JSON 数据存储在 `data/store.json`，可直接备份该文件
