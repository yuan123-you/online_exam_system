## 一、项目概况

### 1.1 项目名称

在线考试系统（Online Exam System）

### 1.2 项目定位

面向高校课程考核场景的在线考试平台，实现管理员、教师、学生三类角色的协同闭环。系统覆盖题库建设、组卷发布、在线答题、自动阅卷、成绩分析、错题复盘全流程。

### 1.3 核心指标

| 指标 | 数据 |
|------|------|
| 角色数 | 3（管理员、教师、学生） |
| 数据表 | 18 张（9 核心 + 9 扩展） |
| API 接口 | 68+ 个（15 个 Controller） |
| Vue 视图 | 25+ 个 |
| Vue 组件 | 18+ 个 |
| 后端 Service | 18 个 |
| 后端 Repository | 9 个 |
| 题库数据 | 1000+ 道 |
| 综合试卷 | 11 套 |
| 测试用例 | 38+ 项（含 147 项后端单元测试） |
| 文档数量 | 12 份 + README |

### 6.1 设计原则

- 所有弹窗基于 BaseModal.vue 基座，统一关闭按钮和布局
- TypeScript 类型定义集中在 types.ts，API 请求封装在 api/client.ts
- ECharts 图表通过 ChartCard.vue 封装，支持 ResizeObserver 自适应
- 批量导入组件自研 xlsx 解析器（ZIP + XML），无第三方依赖

### 6.2 样式设计

- 色调：蓝绿/翡翠绿（Teal-Emerald）为主色，柔和渐变背景
- 卡片：毛玻璃效果（backdrop-filter），悬浮阴影
- 动画：登录页入场动画、弹窗出入场动画、按钮悬浮效果
- 表格：条纹行、hover 高亮、固定表头
- 响应式：三档断点（1180px、960px、640px），支持打印样式
- 可访问性：ARIA 标签、焦点管理、语义化 HTML

## 十、开发规范

### 10.1 前端规范

- Vue 组件使用 `<script setup lang="ts">` 语法
- 类型定义集中在 `types.ts`，避免内联 any
- API 请求统一通过 `api/client.ts` 的 `request` 函数
- 格式化函数集中在 `utils/format.ts`
- CSS 变量统一在 `:root` 定义，组件不硬编码颜色

### 10.2 后端规范

- ApiController 统一处理 HTTP 请求和响应
- StoreService 负责所有数据库操作（JdbcTemplate）
- JSON 字段使用 MySQL JSON 类型存储
- INSERT 语句使用 ON DUPLICATE KEY UPDATE 实现幂等

### 10.3 数据库规范

- 主键使用 VARCHAR(64) 字符串 ID
- JSON 字段以 `_json` 后缀命名
- 外键关系使用逻辑外键（无 FOREIGN KEY 约束）
- 时间字段使用 DATETIME(3) 毫秒精度

## 十一、安全加固措施

项目在代码审计中发现多项安全隐患并完成系统性加固：

### 11.1 密码安全

- 引入 BCrypt 密码哈希（`BCryptPasswordEncoder`），新创建/批量导入/重置的用户密码均以 BCrypt 存储
- 登录时透明兼容旧明文密码：自动检测明文格式并在首次成功登录后升级为 BCrypt 哈希
- 密码修改和重置均使用哈希存储，禁止明文落库

### 11.2 HTTP 安全头

- 新增 `SecurityHeadersFilter` 全局过滤器，为所有响应添加安全头：
  - `X-Content-Type-Options: nosniff` — 阻止 MIME 类型嗅探
  - `X-Frame-Options: DENY` — 防止点击劫持
  - `X-XSS-Protection: 1; mode=block` — 浏览器 XSS 防护
  - `Referrer-Policy: strict-origin-when-cross-origin` — 限制引用泄露
  - `Cache-Control: no-store, no-cache, must-revalidate` — 禁止敏感页面缓存
  - `Content-Security-Policy` — 限制脚本和资源加载源

### 11.3 前端 XSS 防护

- 全面审计并修复 `app.js` 中约 90 处跨站脚本（XSS）漏洞
- 所有渲染函数中的用户可控数据（用户名、题目、班级名等）统一使用 `escapeHtml()` / `escapeAttr()` 转义
- 覆盖 33 个函数：`renderDashboard`、`renderUserSection`、`renderQuestions`、`openEditUser`、`openEditQuestion`、`renderExamModal` 等
- 修复 HTML 属性注入风险：表单 `value` 属性、`data-*` 属性中的用户数据均使用 `escapeAttr()` 处理

### 11.4 CORS 配置

- 新增 `WebConfig` 类配置跨域策略，允许 Vite 开发服务器（端口 5173）安全访问 API

### 11.5 前端体验安全

- 考试进行中添加 `beforeunload` 保护，防止意外关闭导致答卷丢失
- Escape 键关闭模态框，保留考试会话状态
