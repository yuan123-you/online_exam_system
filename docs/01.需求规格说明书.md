# 01.需求规格说明书

## 1. 项目背景与目标

在线考试系统面向高校课程考核场景，目标是实现管理员、教师、学生三类角色的协同闭环，解决传统线下考试组织成本高、阅卷效率低、成绩分析滞后等问题。

系统采用 Spring Boot + Vue 3 + MySQL 前后端分离架构，前端同时保留了基于原生 HTML/CSS/JavaScript 的独立版本，兼顾课程演示和后续扩展需要。

## 2. 用户角色定义

- 管理员：负责账号维护、班级/院系基础数据维护、全局统计与系统日志查看
- 教师：负责题库建设、组卷、发布考试、阅卷和成绩分析
- 学生：负责查看待考考试、在线作答、查询成绩和错题复盘

## 3. 功能需求

### 3.1 管理员端

- 数据总览：查看学生数、教师数、题库量、考试场次等全局统计
- 学生管理：新增、编辑、删除学生账号，支持 Excel/CSV 批量导入
- 教师管理：新增、编辑、删除教师账号，支持批量导入
- 班级管理：维护班级信息（名称、专业、所属学院）
- 学院管理：维护学院组织架构
- 系统日志：查看关键操作记录，按时间倒序展示

### 3.2 教师端

- 教学看板：总览题目数量、试卷数量、考试场次、答卷数等教学数据
- 题库管理：维护题目，支持单选、多选、判断、填空、简答、编程六种题型
- 试卷管理：手动选择题目组成试卷，支持科目和关键字筛选
- 考试管理：选择试卷、班级、考试时间并发布
- 阅卷中心：客观题自动阅卷，主观题人工评分
- 成绩分析：通过 ECharts 图表展示成绩分布和考试通过率

### 3.3 学生端

- 待考考试：查看可参加的考试列表，一键进入考试
- 在线考试：倒计时答题、题号导航、自动保存、单选/多选/判断/文本多种作答方式
- 防作弊提醒：监听 visibilitychange 事件，记录切屏次数，超限自动交卷
- 考试记录：查看历史成绩、排名、用时等详细信息
- 错题本：自动归集错题，支持错题重做和移除
- 个人信息：修改密码

## 4. 非功能需求

- 性能：支持课程项目演示规模下的多用户并发访问，MySQL 持久化存储
- 安全：基于请求头 Authorization Bearer Token 进行身份认证，分角色隔离功能入口
- 易用性：界面采用清新的蓝绿色调设计，支持响应式布局，移动端可自适应浏览
- 可维护性：前后端职责清晰，Spring Boot RESTful API + Vue 3 组件化开发
- 数据一致性：核心数据统一存储在 MySQL，支持 ON DUPLICATE KEY UPDATE 幂等操作

## 5. 界面原型说明

- 登录页：居中卡片式布局，渐变背景，简洁的账号密码表单
- 管理控制台：左侧导航菜单（含品牌信息和用户卡片），右侧业务面板
- 考试页：左侧题号导航和倒计时，右侧题目答题区
- 弹窗组件：统一使用 BaseModal 基座，支持表单编辑、预览、阅卷等场景

## 6. 技术架构概述

- 前端：Vue 3 + TypeScript + Vite（开发端口 5173），ECharts 图表库
- 后端：Spring Boot 3.3.5 + Java 21 + JDBC
- 数据库：MySQL 8.x，数据库名 online_exam_system
- 独立版：Node.js HTTP Server + JSON 文件存储（兼容保留）
- 构建工具：Maven（后端）、Vite（前端）
