# 01.需求规格说明书

## 1. 项目背景与目标

在线考试系统面向学校课程考核场景，目标是实现管理员、教师、学生三类角色的协同闭环，解决传统线下考试组织成本高、阅卷效率低、成绩分析滞后等问题。

## 2. 用户角色定义

- 管理员：负责账号维护、班级/院系基础数据维护、全局统计与系统日志查看
- 教师：负责题库建设、组卷、发布考试、阅卷和成绩分析
- 学生：负责查看待考考试、在线作答、查询成绩和错题复盘

## 3. 功能需求

### 3.1 管理员端

- 学生管理：新增、编辑、删除学生账号
- 教师管理：新增、编辑、删除教师账号
- 班级/院系管理：维护基础组织架构
- 系统日志：查看关键操作记录
- 数据统计：查看用户数、考试场次、参与人次

### 3.2 教师端

- 题库管理：维护题目，支持单选、多选、判断、填空、简答
- 试卷管理：手动选择题目组成试卷
- 考试管理：选择试卷、班级、考试时间并发布
- 阅卷中心：自动阅卷客观题，主观题人工评分
- 成绩分析：统计最高分、最低分、平均分

### 3.3 学生端

- 待考考试：查看可参加的考试列表
- 在线考试：倒计时答题、题号导航、自动保存
- 防作弊提醒：记录切屏次数，超限自动交卷
- 考试记录：查看历史成绩、提交状态
- 错题本：自动归集错题
- 个人信息：修改密码

## 4. 非功能需求

- 性能：支持课程项目演示规模下的多用户访问
- 安全：分角色隔离功能入口，敏感密码不在界面明文展示
- 易用性：界面简洁，移动端可自适应浏览
- 可维护性：前后端职责清晰，核心数据统一存储

## 5. 界面原型说明

- 登录页：左侧项目说明，右侧登录表单
- 管理控制台：左侧菜单，右侧业务面板
- 考试页：左侧题号导航和倒计时，右侧题目答题区
