# 05.项目总结报告

## 1. 开发过程回顾

项目围绕在线考试核心闭环展开，经历了以下阶段：

1. 需求分析：解析原始 PDF 题目说明文档，明确系统功能和角色划分
2. 架构搭建：从 Node.js + JSON 文件的独立版本起步，快速完成核心功能
3. 架构升级：迁移到 Spring Boot + MySQL + Vue 3 前后端分离架构
4. 前端重构：将原生 JavaScript 重构为 Vue 3 组件化架构，引入 TypeScript 类型安全
5. 数据迁移：将 JSON 文件数据转换为 MySQL 建表和 INSERT 脚本
6. 界面优化：设计清新的蓝绿色调主题，优化交互体验和响应式布局
7. 文档整理：同步交付需求规格、系统设计、测试报告、用户手册、部署说明等文档

## 2. 项目亮点

### 2.1 功能完整性

- 实现管理员、教师、学生三端分角色界面，功能入口完全隔离
- 覆盖从题库建设、组卷、发布考试、在线答题、自动阅卷、错题归集的完整闭环
- 集成防切屏提醒与超限自动交卷的防作弊机制
- 支持 Excel/CSV 批量导入学生教师，自研 xlsx 解析器无需第三方依赖
- 集成 AI 智能出题、AI 辅助阅卷、AI 刷题练习（含流式 SSE 对话）三大 AI 功能
- 支持 Excel 成绩导出（Apache POI），及格/不及格颜色标记
- 实现站内通知系统，教师/管理员可发布，支持角色和班级定向推送
- 暗色模式三档切换（浅色/深色/跟随系统），CSS 变量驱动

### 2.2 技术架构

- 前后端分离：Spring Boot RESTful API + Vue 3 组件化
- 类型安全：TypeScript 全面覆盖前端数据类型
- 数据可视化：ECharts 集成成绩分析和题库分布图表
- 双版本兼容：同时提供 Vue 3 现代版和 Spring Boot 内置独立版
- AI 集成：智谱 AI GLM-4.7-Flash 大模型，含熔断器、速率限制、流式 SSE、缓存和降级机制

### 2.3 工程化

- 统一的数据迁移脚本（`scripts/generate-mysql-migration.js`）
- 题库批量生成工具（`scripts/generate-question-bank.js`，1000+ 道题目）
- 烟雾测试验证（`scripts/smoke-test.js`，自动检查数据一致性）
- 完整的文档体系（12 份文档覆盖全部要求）
- 后端单元测试 147 项（JUnit 5 + Mockito），覆盖 11 个核心 Service

## 3. 遇到的问题与解决

| 问题 | 解决方案 |
|------|----------|
| 原始 PDF 仅含题目说明，无源码 | 先解析 PDF 需求，再从零搭建完整演示系统 |
| 主观题难以自动评分 | 设计"客观题自动判分 + 简答题人工阅卷"的混合模式 |
| 中文文本编码异常（乱码） | 开发 normalizeText 工具进行运行时修正 |
| Excel 导入依赖第三方库 | 自研基于 ZIP + XML 的轻量 xlsx 解析器 |
| 前端代码量过大难以维护 | 迁移至 Vue 3 组件化架构，拆分为 18+ 个独立组件和 25+ 个视图 |

## 5. 不足与改进方向

### 5.1 当前不足

- 未实现 JWT Token 认证，当前使用简化的 Bearer Token
- 缺少验证码、操作审计等完整安全能力
- PDF 试卷导出功能尚未实现
- 缺少知识点图谱和更细粒度的教学分析
- 未做深度性能优化和缓存策略

### 5.2 后续改进计划

- 引入 JWT + Spring Security 实现完整认证授权
- 增加 Redis 缓存层提升查询性能
- 增加 PDF 试卷导出
- 引入 WebSocket 实现考试状态实时推送
- 增加知识点关联和智能推荐学习路径
- 引入 Docker Compose 实现一键部署
