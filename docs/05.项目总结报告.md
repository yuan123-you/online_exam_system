# 05.项目总结报告

## 1. 开发过程回顾

项目围绕在线考试核心闭环展开，按照“角色梳理 -> 数据建模 -> 页面实现 -> 考试流程 -> 阅卷分析 -> 文档整理”的顺序推进。由于原始目录仅提供 PDF 题目说明，因此采用从零搭建的方式快速完成课程项目版实现。

## 2. 项目亮点

- 实现管理员、教师、学生三端分角色界面
- 支持从题库、组卷、发布到学生答题、自动阅卷、错题归集的闭环
- 集成防切屏提醒与超限自动交卷
- 同步交付数据库脚本、测试报告、用户手册与部署说明

## 3. 遇到的问题与解决

- 问题：题目需求只在 PDF 中，没有源码基础
  解决：先解析 PDF，再按课程项目目标重建完整演示系统
- 问题：主观题难以自动评分
  解决：设计“客观题自动判分 + 简答题人工阅卷”的混合模式

## 4. 不足

- 当前版本使用 JSON 文件存储，不适合高并发生产环境
- 未接入真正的 Excel 导入导出
- 界面原型仍以课程演示为主，未做更深的视觉打磨

## 5. 后续改进计划

- 升级为 Spring Boot + MySQL + Redis 架构
- 增加 Excel 批量导入、成绩导出
- 引入 JWT、验证码、操作审计等更完整的安全能力
- 增加知识点图谱与更细颗粒度教学分析
