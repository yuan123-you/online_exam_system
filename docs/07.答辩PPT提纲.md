# 07.答辩PPT提纲

## 1. 封面页

- 项目名称：在线考试系统
- 学号/姓名/班级/日期

## 2. 项目背景（1页）

- 在线考试数字化转型需求
- 传统考试的效率和管理痛点：组织成本高、阅卷效率低、成绩分析滞后
- 目标：三端协同的在线考试全流程闭环

## 3. 系统目标（1页）

- 管理员：账号维护、组织架构、全局统计
- 教师：题库建设、组卷发布、阅卷分析
- 学生：在线答题、成绩查询、错题复盘

## 4. 技术架构（1页）

- 前端：Vue 3 + TypeScript + Vite + ECharts
- 后端：Spring Boot 3.3.5 + Java 21 + JDBC
- 数据库：MySQL 8.x
- 架构图：前后端分离，RESTful API 通信
- 备选方案：Node.js + 原生 HTML 独立版

## 5. 核心数据模型（1页）

- 9 张核心数据表：department、class_info、user_account、question、paper、exam、submission、wrong_book_entry、system_log
- 关键关系：院系-班级-学生、教师-题库-试卷-考试-答卷

## 6. 核心功能展示（3-4页）

### 管理员端
- 数据总览看板
- 学生/教师管理（含批量导入）
- 学院/班级组织管理

### 教师端
- 题库管理（6种题型）
- 试卷组卷
- 考试发布与管理
- 阅卷中心（自动+人工）
- 成绩分析图表

### 学生端
- 在线考试（倒计时、自动保存、防作弊）
- 考试记录查询
- 错题本重做

## 7. 项目亮点（1页）

- 防切屏自动交卷机制
- 客观题自动阅卷 + 主观题人工评分
- 自研 xlsx 解析器（零依赖）
- 1000+ 道题库数据
- Vue 3 + TypeScript 组件化前端
- 蓝绿色调现代 UI 设计
- 完整文档体系

## 8. 测试与部署（1页）

- 33 项功能测试用例，全部通过
- 本地启动步骤（MySQL → Spring Boot → Vite）
- 演示账号展示

## 9. 总结与展望（1页）

- 当前成果：完整闭环、双版本、文档齐全
- 已知不足：认证简化、缺少缓存层
- 后续方向：JWT + Spring Security、Redis 缓存、Docker 部署、成绩导出
