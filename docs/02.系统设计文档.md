# 02.系统设计文档

## 1. 总体架构

系统采用轻量三层结构：

- 表现层：`public/index.html + styles.css + app.js`
- 业务层：`server.js` 中的接口路由、评分逻辑、日志记录、权限判断
- 数据层：`data/store.json` 持久化演示数据，`database.sql` 作为关系型数据库设计交付物

## 2. 技术栈

- 前端：原生 HTML、CSS、JavaScript
- 后端：Node.js HTTP Server
- 存储：JSON 文件
- 部署：本地 Node 运行

## 3. 核心数据表设计

- `department`：院系信息
- `class_info`：班级信息
- `user_account`：管理员、教师、学生统一账号表
- `question`：题库表
- `paper`：试卷表
- `exam`：考试发布表
- `submission`：答卷表
- `system_log`：系统日志表

## 4. 核心关系说明

- 一个院系对应多个班级
- 一个班级对应多个学生
- 一个教师可维护多道题目、多个试卷和考试
- 一个考试绑定一套试卷并面向多个班级
- 一个学生对一个考试形成一份答卷

## 5. 接口设计

- `POST /api/login`：登录
- `GET /api/bootstrap`：获取全量初始化数据
- `POST /api/entities`：通用新增
- `PUT /api/entities/users`：编辑用户
- `DELETE /api/entities/:entity/:id`：删除数据
- `GET /api/exams/:id/detail`：查看考试明细
- `POST /api/submissions/save`：保存答卷草稿
- `POST /api/submissions/submit`：提交答卷并自动阅卷
- `POST /api/submissions/manual-grade`：教师人工阅卷
- `POST /api/user/password`：修改密码

## 6. 关键模块设计

### 6.1 权限控制

通过请求头 `X-User-Id` 识别当前用户，在服务端校验角色权限并限制接口访问。

### 6.2 自动阅卷

- 单选/判断：直接比较单值
- 多选：排序后比较集合
- 填空：忽略大小写后比较文本
- 简答：进入待阅卷状态，由教师人工评分

### 6.3 防作弊机制

- 监听浏览器 `visibilitychange`
- 记录切屏次数
- 超过考试配置阈值时自动交卷并标记为可疑答卷

### 6.4 自动保存

学生可手动保存，切题时同步维护当前作答状态，后端持久化到答卷草稿。
