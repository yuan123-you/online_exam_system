# 02.系统设计文档

## 1. 总体架构

系统采用前后端分离的三层结构：

- 表现层：Vue 3 组件化前端（`src/`），以及独立版原生 HTML/CSS/JavaScript（`public/` 和 `backend/src/main/resources/static/`）
- 业务层：Spring Boot RESTful API（`backend/`），接口路由、评分逻辑、日志记录、权限判断
- 数据层：MySQL 关系型数据库，`database.sql` 提供完整的建库建表和数据初始化脚本

## 4. 核心关系说明

- 一个院系对应多个班级（1:N）
- 一个班级对应多个学生（1:N）
- 一个教师可维护多道题目、多个试卷和考试（1:N）
- 一个考试绑定一套试卷并面向多个班级（1:1, 1:N）
- 一个学生对一个考试形成一份答卷（1:1 UNIQUE）
- 一个学生可有多条错题记录（1:N）

## 6. 关键模块设计

### 6.1 权限控制

通过请求头 `Authorization: Bearer {userId}` 识别当前用户，在 ApiController 中解析并校验角色权限，不同角色的 bootstrap 数据返回不同过滤结果。

### 6.2 自动阅卷

- 单选/判断：直接比较单值
- 多选：排序后比较集合
- 填空：忽略大小写后比较文本
- 简答/编程/代码填空：进入待阅卷状态，由教师人工评分

### 6.3 防作弊机制

- 监听浏览器 `visibilitychange` 事件
- 记录切屏次数并自动保存
- 超过考试配置阈值时自动交卷并标记为可疑答卷

### 6.4 自动保存

学生可手动保存答卷，切题时同步维护当前作答状态，系统每 30 秒自动保存一次草稿到后端。
